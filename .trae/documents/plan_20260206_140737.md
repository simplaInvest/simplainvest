# Diagnóstico da Seção de Analytics

A seção de Analytics no gerador de Debriefing utiliza a API do **Google Analytics 4 (GA4)** para buscar dados de tráfego e conversão. Abaixo está o detalhamento técnico e os pontos de atenção que podem explicar a discrepância nos números.

## 1. Como funciona tecnicamente

O sistema não lê os dados de "Conversões" configurados dentro do painel do GA4 (Eventos de Conversão). Em vez disso, ele faz uma **inferência baseada em visitas a páginas específicas**.

* **Fonte de Dados:** API do Google Analytics 4 (`google.analytics.data_v1beta`).

* **Métrica Utilizada:** `totalUsers` (Total de Usuários Únicos).

  * *Nota:* O sistema conta **pessoas únicas**, não sessões ou visualizações de página. Se uma pessoa visita a página 5 vezes, conta como 1.

* **Definição de "Visita":** Número de usuários únicos que acessaram uma URL contendo um trecho específico (ex: `/cursogratuito`).

* **Definição de "Conversão":** Número de usuários únicos que acessaram a URL de agradecimento (ex: `/cg/inscricao-pendente`).

* **Cálculo da Taxa:** `(Usuários na Página de Obrigado / Usuários na Página de Captura) * 100`.

## 2. Pontos de Atenção (Por que os números podem parecer errados)

Identifiquei 4 causas prováveis para estranheza nos números:

### A. URLs "Chumbadas" no Código (Hardcoded)

O código busca por trechos de URL fixos. Se a equipe de marketing mudou a URL da página, o sistema parou de contar ou está contando errado.

* **Captura (EI):** Busca por `/cursogratuito`

* **Obrigado (EI):** Busca por `/cg/inscricao-pendente`

* **Pré-Matrícula (EI):** Busca por `/ei/prematricula` e `/parabens-ei`

* **Vendas (EI):** Busca por `/euinvestidor` e `/?X11050759S` (Hotmart)

> **Risco:** Se a página de vendas mudou de `/euinvestidor` para `/euinvestidor-v2`, e o código busca apenas `/euinvestidor`, ele pode ignorar a nova página ou misturar dados.

### B. Filtro de URL "Abrangente" (Regex)

O sistema usa um filtro do tipo "Contém" (`PARTIAL_REGEXP`).

* Se o filtro é `/cursogratuito`, ele vai somar dados de:

  * `simplainvest.com/cursogratuito` (Correto)

  * `simplainvest.com/cursogratuito-antigo` (Provavelmente Incorreto)

  * `simplainvest.com/teste-cursogratuito` (Incorreto)
    Isso tende a **inflar** o número de visitas se houver páginas de teste ou legadas com nomes parecidos no mesmo período.

### C. Janela de Datas

As datas são puxadas da planilha de lançamentos (`DF_CENTRAL_LANCAMENTOS`).

* Se a data de início/fim na planilha estiver deslocada em 1 ou 2 dias em relação ao disparo real dos e-mails/anúncios, você pode estar vendo um período com tráfego residual ou incompleto.

### D. Usuários vs. Sessões

Muitos gestores de tráfego olham para "Sessões" ou "Cliques no Link". O dashboard olha para "Usuários Únicos".

* Geralmente, `Usuários < Sessões < Cliques`.

* Se você comparar com o Gerenciador de Anúncios (Facebook Ads), o número do GA4 (Usuários) será naturalmente menor.

## Próximos Passos Sugeridos

Para corrigir ou validar, recomendo:

1. **Verificar as URLs atuais:** Confirmar se as URLs usadas no lançamento correspondem exatamente aos trechos listados acima.
2. **Validar as Datas:** Conferir na aba "Geral" ou na planilha fonte se as datas de Captura/Vendas estão precisas.
3. **Ajuste de Código (Se necessário):** Podemos alterar o código para usar "Sessões" em vez de "Usuários" se preferir alinhar com a visão de tráfego, ou refinar o filtro de URL para ser exato (`EXACT`) ao invés de parcial.
